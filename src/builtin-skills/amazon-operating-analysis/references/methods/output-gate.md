# 输出门禁规则

## 必查项

| 检查项 | 要求 |
|---|---|
| 结论等级 | 必须写明正式测算、暂估测算、方向性提示或资料缺口清单。 |
| 资料缺口 | M0/M1 缺口必须可见。 |
| 假设披露 | 所有默认值、M1/M2 假设和 AI 暂估都必须披露。 |
| 计算与证据来源 | 关键数字要指向文件、用户陈述、模型默认或 `operating_analysis_calculate` 输出。 |
| 风险提示 | 话术不能比证据更确定，高风险变量必须单列。 |
| 顾问复核边界 | 明确这是顾问底稿/内部辅助，不是未经复核的正式客户结论。 |
| 受众 | 顾问底稿、客户摘要、GUI 状态和报告初稿的话术不同。 |

## 话术规则

暂估测算使用“基于当前假设”；方向性提示使用“方向上”；资料缺口清单不输出量化经营结论。

每次最终输出固定包含六个可见小节：`结论等级`、`资料缺口`、`假设披露`、`计算与证据来源`、`风险提示`、`顾问复核边界`。缺 M0 时也必须保留这些小节，其中“顾问复核边界”说明当前内容只能作为内部沟通或资料清单，不能作为正式对外经营结论。

资料完整但仍有税费、合规、回款、reserve、库存成本等边界时，`资料缺口` 不得省略，应写“无 M0 阻塞，仍需复核的 M1/M3 项：...”。无法归因费用必须单独披露，不得硬摊到 SKU 后包装成正式复盘。

## 经营分析硬门禁

- 缺售价、采购成本、站点/履约方式时，不输出立项测算结论。
- 缺期初现金、回款周期或采购付款节点时，不输出“钱够不够”的正式结论。
- 使用 Excel 默认值时，结论最高为暂估测算，且默认值必须进入假设表。
- 税费、认证、合规、平台冻结款、长期仓储费、季节性、断货影响不得沉默默认。
- calculator 输出只能作为顾问底稿、客户摘要和报告/PPT 初稿素材；正式对外内容仍需顾问复核。
