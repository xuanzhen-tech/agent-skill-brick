# 资料索要规则

## 资料请求结构

资料请求应包含：

1. 当前要回答的业务问题。
2. 现在必须补齐的 M0 资料。
3. 可以暂估的 M1 资料。
4. 可以披露后使用的 M2 默认值。
5. 缺失后会阻塞或降级哪个结论。

## 最小范围字段

| 字段 | 作用 |
|---|---|
| 分析目标 | 决定主责技能。 |
| 平台、站点、店铺 | 决定费率和回款假设。 |
| 期间或预测周期 | 区分历史复盘、未来预测和现金流。 |
| 币种 | 避免混币种输出。 |
| SKU、产品或项目粒度 | 决定模型粒度。 |
| 输出对象 | 决定话术和输出深度。 |

## 对话规则

不要要求用户“上传所有相关资料”。每次只索要能让当前结论提升一个等级的最小资料集。
