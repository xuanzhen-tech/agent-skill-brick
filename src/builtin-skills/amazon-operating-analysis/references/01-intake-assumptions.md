# 资料与假设管理

## 触发

当经营目标、范围、必需资料、假设来源或结论等级不清时使用。

## 边界

只负责资料索要和假设控制，不负责单件利润、销售预测、现金流诊断或最终经营建议。

## 加载下游

- 范围和 M0 缺口明确后，回到 `references/00-router.md`。
- 必需资料状态清楚后，再加载主责业务模块。
- 对外输出前加载 `references/methods/output-gate.md`。

## 必读资料

- `references/shared/material-levels-and-conclusion-levels.md`
- `references/shared/assumption-register-policy.md`
- `references/shared/default-value-policy.md`
- `references/shared/intake-material-request-rules.md`

## 停止与降级

主责问题的 M0 资料缺失时，只输出资料缺口和下一步索要话术，不输出正式经营结论。
