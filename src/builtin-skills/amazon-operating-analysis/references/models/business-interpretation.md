# 顾问解释规则

## 解释顺序

1. 先说明结论等级。
2. 再回答用户真实问题。
3. 展示关键数字和最大驱动因素。
4. 披露假设和资料缺口。
5. 给下一步资料请求或顾问复核动作。

## 常见问题解释

| 用户问题 | 应解释 |
|---|---|
| 这个产品能不能做 | 利润空间、启动资金压力、现金流风险、最敏感变量、缺口资料。 |
| 卖多少钱合适 | 单件贡献利润、可承受 ACoS、保本销量、竞品价格是否未纳入。 |
| ACoS 到多少会亏 | 当前 ACoS、亏损边界、广告销售口径、广告出单占比假设。 |
| 30万够不够 | 启动费用、首批采购、回款延迟、最低现金余额、风险月份。 |
| 报表盈利但没钱 | 收入确认与回款错位、库存占用、采购付款、平台 reserve、仓储费。 |

## 话术边界

- 不说“建议一定做/不做”，改说“按当前假设，顾问可重点关注”。
- 不说“资金一定够”，改说“在当前回款和付款假设下，最低现金余额为...”。
- 不说“定价应为 X”，改说“若目标贡献利润率为 Y，售价需不低于 X”。
- 不说“库存应立即清理”，改说“这些 SKU 需要优先复核清货可能性”。
