# 库存风险规则

## 资料需求

| 变量 | 用途 | 等级 | 可否假设 | 默认来源 | 缺失影响 | 索要话术 |
|---|---|---|---|---|---|---|
| 库存范围和地点 | 区分 FBA、海外仓、在途、工厂未提货、不可售 | M0 | 否 | 用户、库存表 | 无法汇总库存 | 请确认纳入哪些库存地点和状态。 |
| 库存数量和状态 | 计算可售、不可售、预留 | M0 | 否 | 库存快照、分类账 | 库存事实阻塞 | 请提供库存快照或 Amazon 库存分类账。 |
| 单位成本口径 | 计算库存金额 | M0/M1 | 不宜沉默 | 采购、头程、ERP | 金额降级 | 请确认库存成本包含采购、头程、关税还是只含采购。 |
| 库龄起算日 | 判断滞销和长期仓储 | M1 | 可暂按入仓日 | 库龄报表 | 滞销判断降级 | 库龄按采购、入仓、上架还是可售日期起算？ |
| 动销窗口 | 判断周转 | M1 | 可场景化 | 历史销售 | 周转天数降级 | 请确认用近 7、30、60 还是 90 天销量判断动销。 |
| 仓储费和长期仓储费 | 判断利润侵蚀 | M1 | 不宜沉默 | 仓储费报表 | 利润影响降级 | 请提供月仓储费、长期仓储费或费率。 |
| 在途和未提货付款状态 | 关联现金占用 | M1 | 可暂估 | 采购、物流 | 现金占用降级 | 在途和工厂未提货是否已经付款？ |

## 假设政策

- 库存成本口径、可售/不可售、长期仓储费、在途是否已付款、库龄阈值不能沉默假设。
- 不硬编码 60/90/180 天阈值；没有确认时，阈值进入假设表。
- 库存资金占用先算事实，不直接替代全局现金流预测。
- 清货建议必须说明利润损失、现金回收和继续持有成本。

## 计算规则

| 输出指标 | 公式原型 | 输入变量 | 适用边界 |
|---|---|---|---|
| 库存占用金额 | 库存数量 * 单位成本 | 数量、成本 | 成本口径必须披露。 |
| 可售库存天数 | 可售数量 / 日均销量 | 可售库存、动销窗口 | 断货或促销会扭曲。 |
| 周转天数 | 库存金额 / 日均销售成本 | 库存金额、销售成本 | 适合成熟 SKU。 |
| 库龄结构 | 各库龄段库存 / 总库存 | 库龄、数量 | 阈值需确认。 |
| 仓储费侵蚀率 | 仓储费 / 销售额或毛利 | 仓储费、销售、毛利 | 用于风险解释。 |
| 清货优先级 | 库龄、周转、毛利、仓储费、现金占用综合排序 | 多指标 | 是管理建议，不是自动处置。 |
| 清货动作比较 | 继续持有、低价清仓、Outlet/站外 Deal、移除、销毁的现金回收和损失 | 成本、售价、清仓价、售罄率、移除费、销毁费、仓储费 | 是处置底稿，不是自动执行。 |

## 门禁降级

| 资料状态 | 可输出 | 不可输出 |
|---|---|---|
| 缺库存数量 | 资料缺口清单 | 库存金额、滞销清单 |
| 缺成本 | 库存数量和库龄 | 库存资金占用金额 |
| 缺动销 | 库存金额 | 周转和滞销判断 |
| 数量、成本、动销、库龄齐全 | 正式库存风险分析 | 未披露阈值的清货承诺 |

## 输出模板

顾问底稿：库存范围、数量、成本口径、库龄、动销、仓储费、风险分级、清货优先级和资料缺口。

客户摘要：说明钱压在哪些 SKU、哪些库存滞销、仓储费如何侵蚀利润、优先处理顺序。

GUI 状态：显示库存事实是否齐全、成本口径是否确认、风险 SKU 数量和结论等级。

报告初稿：按“库存金额、库龄结构、滞销清单、仓储费影响、清货建议、风险边界”组织。

## 真实对话执行模板

- 用户问“库存压了多少钱”时，必须先确认库存数量、地点/状态和单位成本口径；缺成本时不得输出正式库存金额。
- 用户问“哪些 SKU 要清”时，必须同时看库龄、动销、毛利/仓储费和可售状态；缺动销或毛利时降级，不给正式清货优先级。
- 用户问“仓储费怎么侵蚀利润”时，保留结论等级、资料缺口和复核边界；缺单件毛利或销售额时只能输出方向性提示。
- 用户问“降价、移仓、移除还是销毁”时，必须调用 `calculate_slow_moving_inventory_decision`，比较继续持有、低价清仓、Outlet/站外 Deal、移除和销毁，不得只给一句“建议清货”。
- 读取真实库存 ledger 或 outputs 时，要在计算与证据来源小节列明文件名和口径，不能未读取文件却声称基于真实数据。
- 最终回答固定六小节：结论等级、资料缺口、假设披露、计算与证据来源、风险提示、顾问复核边界。

## Calculator 交接

主 calculator：`calculate_inventory_capital_risk`。

滞销处置 calculator：`calculate_slow_moving_inventory_decision`。

调用条件：

- 已拿到库存数量、地点、状态，至少能形成 SKU 级库存事实。
- 用户问题指向库存占用金额、库龄结构、滞销清单、仓储费影响或清货优先级。
- 用户问题指向仓储费侵蚀、清仓、移除、销毁、Outlet/站外 Deal 或清货动作比较时，调用 `calculate_slow_moving_inventory_decision`。
- 有单位成本时输出库存占用金额；有动销窗口销量时输出滞销和周转判断。
- 真实 workspace 只有库存数量和仓储费、缺成本/动销/库龄时，也必须调用 `calculate_slow_moving_inventory_decision`，让 calculator 以方向性提示返回清货动作缺口；不得用“缺数据”绕过 calculator。

滞销处置最小输入样板：

```json
{
  "items": [
    {
      "sku": "示例 MSKU",
      "inventory_units": 1000,
      "monthly_storage_fee_cny": 200,
      "unit_cost_cny": null,
      "unit_contribution_cny": null,
      "age_days": null,
      "sales_30d_units": null,
      "clearance_price_cny": null,
      "remove_fee_cny": null,
      "dispose_fee_cny": null
    }
  ],
  "holding_months": 3,
  "missing_materials": ["单位成本", "近30/60/90天销量", "库龄", "清仓价/移除费/销毁费"]
}
```

禁止条件：

- 缺库存数量时禁止调用 calculator 输出经营结论，只能索要库存快照或库存 ledger。
- 缺成本时禁止输出库存资金占用金额，只能输出数量、地点、状态和库龄方向性提示。
- 缺动销时禁止输出正式滞销结论和清货优先级，只能说明需要补销量窗口。
- 缺单件毛利、清仓价格、移除费或销毁费时，滞销处置只能降级为方向性提示。

顾问复核边界：

- 清货优先级是顾问底稿，不是自动处置指令。
- 成本口径、可售/不可售、长期仓储费、在途是否已付款必须由安信或客户确认。
- 清货动作需要复核亏损折扣、现金回收、继续持有仓储费和账号运营风险。

## 样例

| 场景 | 输入状态 | 输出等级 | 输出重点 |
|---|---|---|---|
| 资料完整 | 有库存、成本、库龄、动销、仓储费 | 正式测算 | 资金占用、滞销清单和清货优先级。 |
| 资料不足 | 有数量无成本 | 方向性提示 | 只能看数量和库龄，不能看占用金额。 |
| 高风险假设 | 库龄阈值未确认 | 暂估测算 | 披露阈值变化对滞销清单的影响。 |
