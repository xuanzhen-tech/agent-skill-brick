# 未来经营预测规则

## 资料需求

| 变量 | 用途 | 等级 | 可否假设 | 默认来源 | 缺失影响 | 索要话术 |
|---|---|---|---|---|---|---|
| 预测周期和粒度 | 决定输出表 | M0 | 否 | 用户 | 无法预测 | 请确认预测未来几周或几个月，以及按 SKU、产品线还是店铺预测。 |
| 历史销售基线 | 建立起点 | M0/M1 | 可被计划替代 | 历史销售、利润表 | 预测降级 | 请提供近 3 到 12 个月销量、销售额或已复盘结果。 |
| 增长率或经营目标 | 生成场景 | M0 | 必须披露 | 用户计划 | 无法生成场景 | 请给出保守、基准、乐观增长假设，或说明目标销量。 |
| 价格和促销计划 | 影响销售额和毛利 | M1 | 可场景化 | 用户计划 | 利润降级 | 未来是否有提价、降价、优惠券或大促？ |
| 广告预算和效率 | 预测广告和利润 | M1 | 可场景化 | 历史广告、计划 | 利润降级 | 请提供广告预算、目标 ACoS 或历史广告效率。 |
| 库存和补货约束 | 限制销量 | M1 | 不宜忽略 | 库存表、采购计划 | 销量偏乐观 | 请提供当前库存、在途和补货计划。 |
| 上新、断货、季节性 | 调整基线 | M1 | 必须披露 | 运营计划 | 趋势失真 | 是否有上新、断货、旺季淡季或活动计划？ |
| 促销提升和断货损失率 | 修正可实现销量 | M1 | 可场景化 | 运营计划、历史活动 | 销量偏差 | 促销预计提升多少？断货或限库存会损失多少需求？ |

## 假设政策

- 未来预测必须区分历史事实、用户计划和 AI 暂估。
- 增长率、季节性、促销、断货、上新和广告效率变化不能沉默假设。
- 没有历史基线但有用户计划时，可以做计划测算，不能称为历史趋势预测。
- 预测结果可输入现金流预测，但不能直接回答现金是否够用。

## 计算规则

| 输出指标 | 公式原型 | 输入变量 | 适用边界 |
|---|---|---|---|
| 销量预测 | 历史基线 * 增长系数 * 季节性 * (1 + 促销提升) * (1 - 断货损失) + 上新计划销量，并受库存约束 | 历史销量、增长、库存、季节性、促销、断货、上新 | 上新无历史时只能场景化。 |
| 销售额预测 | 预测销量 * 价格 | 销量、价格、促销 | 折扣需单列。 |
| 毛利预测 | 销售额 - 变动成本 | 单件模型、销量 | 成本变动需披露。 |
| 广告预算预测 | 销售额 * ACoS 或计划预算 | ACoS、销售额 | 要区分总广告和归因广告。 |
| 采购需求 | 目标库存 - 可用库存 - 在途 + 安全库存 | 销量、库存、补货周期 | 不是付款时间表。 |
| 库存需求 | 预测销量 * 覆盖天数 | 销量、周转目标 | 要和库存风险联动。 |

## 门禁降级

| 资料状态 | 可输出 | 不可输出 |
|---|---|---|
| 缺周期或粒度 | 资料缺口清单 | 任何预测表 |
| 缺历史且缺计划 | 资料请求和预测框架 | 数值预测 |
| 有计划但缺历史 | 计划测算 | 趋势预测 |
| 历史、计划、库存齐全 | 正式或暂估预测 | 未披露假设的确定销量承诺 |

## 输出模板

顾问底稿：预测目标、历史基线、场景假设、销量、销售额、毛利、广告、采购和库存需求。

客户摘要：说明未来几个月经营表现、最可能的压力点、需要提前动作的广告和采购计划。

GUI 状态：显示预测周期、场景版本、库存约束是否纳入和结论等级。

报告初稿：按“历史基线、三档场景、关键变量、库存约束、风险和下一步”组织。

## 库存约束输出要求

当用户提供当前库存、在途或补货计划时，预测输出必须同时写明两组数量：

- `需求预测`：按历史基线、计划或场景推导出的原始需求。
- `可实现销量`：受可售库存、在途、补货周期和断货影响后的销量上限。

如果库存不足，必须明确写出需求缺口、断货时点和“库存约束后销量不得高于可售/可补货能力”。不能只给乐观需求预测。

## Calculator 交接

主 calculator：`calculate_operating_forecast`。

调用条件：

- 用户问题指向未来销量、销售额、利润、广告预算预测、采购需求或库存需求。
- 已确认预测周期和粒度，并且至少有历史基线或用户经营计划。
- 有库存快照、在途或补货计划时，必须作为库存约束输入，不得只按增长率外推。
- 真实 workspace 只有单月历史快照时，仍必须调用 `calculate_operating_forecast`，把已有单月销量/销售额/广告/库存作为 `historical_baseline`，并通过 `missing_materials` 披露“缺连续多月历史”；结论等级降级为资料缺口清单或方向性提示。

最小输入样板：

```json
{
  "forecast_periods": ["M1", "M2", "M3"],
  "granularity": "sku",
  "historical_baseline": [
    {
      "sku": "示例 MSKU",
      "period": "2026-03",
      "units": 100,
      "revenue_cny": 10000,
      "contribution_profit_cny": 2000,
      "ad_spend_cny": 1500,
      "available_inventory_units": 120
    }
  ],
  "scenario_assumptions": {
    "base": { "growth_rate": 0, "target_acos": 0.3 }
  },
  "missing_materials": ["连续 3-6 个月历史销量", "未来促销计划", "补货计划"]
}
```

输出要求：即使不能做正式趋势预测，最终回答也必须写“库存约束”“采购需求”“广告预算”三个小节或表格字段；无法计算时写成资料缺口和复核边界。

最终回答必须显式标注本次属于哪一种：

- `趋势预测`：有连续多期历史基线，并基于历史趋势外推。
- `计划测算`：没有足够历史趋势，但有用户经营目标、促销、广告或补货计划。
- `场景预测`：以保守/基准/乐观假设生成多版本结果。

不要只写“未来判断”或“方向性提示”，必须在结论等级或计算与证据来源中出现上述三个标签之一。

禁止条件：

- 缺周期或粒度时禁止生成数值预测。
- 缺历史且缺计划时禁止输出销量、利润和广告预算预测，只能索要资料。
- 有计划但缺历史时只能称为计划测算，不能称为趋势预测。
- 库存约束缺失时必须披露“销量可能偏乐观”，不得输出确定销量承诺。

顾问复核边界：

- 增长率、季节性、促销、广告效率、断货、上新都必须进入假设披露。
- 当用户只给老板目标销量、没有历史基线时，输出必须写“计划测算”，不能写“趋势预测”。
- 预测结果可输入现金流预测，但不能直接回答“钱够不够”。
- 广告预算预测需要复核 ACoS 是目标、历史均值还是用户预算，不得混用。

## 样例

| 场景 | 输入状态 | 输出等级 | 输出重点 |
|---|---|---|---|
| 资料完整 | 有历史、广告、库存、计划 | 正式测算 | 三个月销量、利润、广告和采购需求。 |
| 资料不足 | 只有目标增长 | 方向性提示 | 要求补历史和库存，先列预测框架。 |
| 高风险假设 | 季节性和广告效率不明 | 暂估测算 | 披露增长率和 ACoS 对利润的影响。 |
